# Документация команд BMCU для Klipper

В данном документе описаны все команды (G-code) и макросы, предоставляемые Klipper-интеграцией для BMCU (через модуль `bmcu.py` и файл конфигурации `bmcu_macros.cfg`). Эти команды предназначены для прямого вызова из консоли Klipper, макросов печати или пользовательских интерфейсов (Mainsail/Fluidd).

---

## 1. Базовые G-code команды (реализованы в `bmcu.py`)

Данные команды отправляют соответствующие пакеты в прошивку BMCU по последовательному порту. Для большинства команд доступен параметр `WAIT`, который указывает время (в секундах) ожидания ответа от платы.

### `BMCU_CAPS`
Выводит список разрешенных команд, если в конфигурации Klipper (`supported_cmds`) задан белый список. Иначе выводит сообщение, что разрешены все команды.
* **Аргументы:** Нет

### `BMCU_PING`
Проверяет связь с платой BMCU.
* **Аргументы:**
  * `WAIT` (Float, по умолчанию 0.0)

### `BMCU_STATUS`
Выводит в консоль Klipper закешированный статус от BMCU (без запроса новых данных у платы).
* **Аргументы:** Нет

### `BMCU_TEST_MOTOR`
Запуск диагностического прогона мотора.
* **Параметры:** `LANE`, `PWM`, `DURATION`.

### `BMCU_SAVE_PARAMS`
Сохранение всех текущих настроек в энергонезависимую память BMCU.
* **Параметры:** `WAIT` (опционально).

### `BMCU_LOAD_PARAMS`
Загрузка настроек из памяти BMCU (откат изменений в RAM).
* **Параметры:** `WAIT` (опционально). (Float, по умолчанию 0.5)

### `BMCU_STOP`
Экстренно останавливает все моторы BMCU.
* **Аргументы:** Нет

### `BMCU_GET_SENSORS`
Запрашивает актуальное состояние датчиков.
* **Аргументы:**
  * `WAIT` (Float, по умолчанию 0.5)

### `BMCU_STOP`
Экстренно останавливает все моторы BMCU.
* **Аргументы:** Нет

### `BMCU_SELECT_LANE`
Выбирает активный канал (смену филамента).
* **Аргументы:**
  * `LANE` (Int, по умолчанию 0): Номер канала (0-3).
  * `WAIT` (Float, по умолчанию 0.3)

### `BMCU_SET_AUTO_FEED`
Включает или выключает режим автоподачи для указанного канала.
* **Аргументы:**
  * `LANE` (Int, по умолчанию 0): Номер канала.
  * `ENABLE` (Int, по умолчанию 1): 1 для включения, 0 для выключения.

### `BMCU_MOVE`
Универсальная команда движения.
* **Аргументы:**
  * `AXIS` (String, по умолчанию "0"): Ось ("0", "1", "2", "3", "FEED").
  * `DIST` (Float, по умолчанию 0.0): Дистанция в мм.
  * `SPEED` (Float, по умолчанию из конфига `default_speed`): Скорость.
  * `WAIT` (Float, по умолчанию 0.0)

### `BMCU_FEED`
Подает филамент в текущем активном канале.
* **Аргументы:**
  * `MM` (Float, по умолчанию из конфига `default_lane_feed_mm`): Дистанция подачи.
  * `SPEED` (Float, по умолчанию `default_speed`): Скорость подачи.


### `BMCU_LANE_FEED` / `BMCU_LANE_RETRACT`
Подача или ретракт филамента на указанном канале без его предварительного выбора (напрямую дергает мотор канала).
* **Аргументы:**
  * `LANE` (Int, по умолчанию 0): Номер канала.
  * `MM` (Float, по умолчанию из конфига `default_lane_feed_mm` / `default_lane_retract_mm`): Дистанция.
  * `SPEED` (Float, по умолчанию `default_speed`): Скорость.


### `BMCU_CALL`
Вызов произвольной JSON-команды прошивки напрямую.
* **Аргументы:**
  * `CMD` (String, обязательно): Имя JSON команды (например, "CALIBRATE").
  * `ARGS` (String): JSON-строка аргументов (например, `"{\"lane\": 0}"`).
  * `WAIT` (Float, по умолчанию 0.0)

## 2. Klipper Макросы (`bmcu_macros.cfg`)

Для удобства пользователя созданы готовые макросы, оборачивающие базовые команды. Они идеально подходят для кнопок в Mainsail/Fluidd.

### Глобальные макросы
* **`BMCU_STOP_ALL`**: Вызов `BMCU_STOP`.
* **`BMCU_SHOW_STATUS`**: Вывод кешированного статуса (`BMCU_STATUS`).
* **`BMCU_FETCH_STATUS`**: Запрос нового статуса у платы (`BMCU_CALL CMD=STATUS WAIT=0.5`).
* **`BMCU_REFRESH`**: Аналог FETCH_STATUS, но с увеличенным временем ожидания (1 секунда).

### Канальные макросы (для каждого канала L0, L1, L2, L3)
Где `X` — номер канала (от 0 до 3).
* **`BMCU_SELECT_LX`**: Выбор канала (например, `BMCU_SELECT_L0`).
* **`BMCU_FEED_LX`**: Подача на канале со значениями по умолчанию.
* **`BMCU_RETRACT_LX`**: Ретракт на канале со значениями по умолчанию.
* **`BMCU_AUTO_ON_LX`**: Включение автоподачи на канале.
* **`BMCU_AUTO_OFF_LX`**: Выключение автоподачи на канале.
