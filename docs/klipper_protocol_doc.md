# Klipper-BMCU Протокол (JSON)

Взаимодействие между пластмассой BMCU370 и Klipper осуществляется по последовательному порту (UART) с использованием текстового JSON-протокола, завершаемого `\r\n`.

Исходный код реализации находится в `src/api/KlipperCLI.cpp`.

## Общий формат

**Запрос (Klipper → BMCU):**
Каждый запрос должен содержать уникальный `id` (число) и команду `cmd`. Параметры передаются внутри вложенного объекта `args` или в корне.
```json
{"id": 101, "cmd": "COMMAND_NAME", "args": { "param1": "value" }}
```

**Ответ Успех (BMCU → Klipper):**
Ответ всегда возвращает тот же `id`, `ok: true`, и может содержать полезную нагрузку.
```json
{"id": 101, "ok": true, "code": "OK", "msg": "Success"}
```

**Ответ Ошибка:**
```json
{"id": 101, "ok": false, "code": "ERR_CODE", "msg": "Error description"}
```

**Событие (без запроса):**
```json
{"event": "STARTUP", "msg": "KlipperCLI Ready"}
```

## Поддерживаемые команды

### 1. `PING`
Проверка связи. Возвращает телеметрию.
* **Запрос:** `{"id": 1, "cmd": "PING"}`
* **Ответ:** `{"id": 1, "cmd": "PING", "ok": true, "result": "ok", "telemetry": {"version": "00.00.05.00", "uptime": 12345}}`

### 2. `STATUS`
Запрос полного состояния всех каналов (Lanes). Именно эту команду использует веб-интерфейс `bmcu_card.html` для отрисовки интерфейса.
* **Запрос:** `{"id": 2, "cmd": "STATUS"}`
* **Ответ:** 
```json
{
  "id": 2,
  "cmd": "STATUS",
  "ok": true,
  "lanes": [
    {
      "id": 0,
      "present": true,
      "motion": "Idle",
      "meters": 120.50,
      "pressure": 1.650,
      "rfid": "A1B2C3D4",
      "name": "PLA Orange",
      "temp_min": 190,
      "temp_max": 220,
      "color": [255, 128, 0, 255]
    },
    // ... еще 3 канала
  ]
}
```

### 3. `GET_SENSORS`
Возвращает массив состояний датчиков филамента (вставлен/нет) для каждого канала.
* **Запрос:** `{"id": 3, "cmd": "GET_SENSORS"}`
* **Ответ:** `{"id": 3, "cmd": "GET_SENSORS", "ok": true, "lane": [1, 0, 0, 1]}` (1 - есть пруток, 0 - пусто)

### 4. `MOVE`
Перемещение мотора на заданное расстояние.
* **Аргументы:** 
  * `axis` (строка) — номер канала `"0"`-`"3"` или `"FEED"` (активный канал).
  * `dist_mm` (число) — дистанция в мм.
  * `speed` (число) — скорость.
* **Запрос:** `{"id": 4, "cmd": "MOVE", "axis": "0", "dist_mm": 50.0, "speed": 10.0}`

### 5. `STOP`
Немедленная остановка всех моторов.
* **Запрос:** `{"id": 5, "cmd": "STOP"}`

### 6. `SELECT_LANE`
Устанавливает активный канал. Выбранный канал будет использоваться при `MOVE` с осью `FEED`.
* **Аргументы:** `lane` (0-3).
* **Запрос:** `{"id": 6, "cmd": "SELECT_LANE", "lane": 1}`

### 7. `SET_AUTO_FEED`
Включает или выключает авто-подачу (ПИД по давлению) для конкретного канала.
* **Аргументы:** `lane` (0-3), `enable` (true/false).
* **Запрос:** `{"id": 7, "cmd": "SET_AUTO_FEED", "lane": 0, "enable": true}`

### 8. `GET_FILAMENT_INFO`
Получить сохраненную во Flash-памяти информацию о катушке в определенном канале.
* **Аргументы:** `lane` (0-3).
* **Запрос:** `{"id": 8, "cmd": "GET_FILAMENT_INFO", "lane": 0}`

### 9. `SET_FILAMENT_INFO`
Записать информацию о катушке в определенный канал (сохраняется во Flash).
* **Аргументы:** 
  * `lane` (0-3) - обязательно
  * `name` (строка, макс 20 символов) - опционально
  * `id_str` (строка, макс 8 символов, для RFID/метки) - опционально
  * `temp_min` (число) - опционально
  * `temp_max` (число) - опционально
  * `color` (массив из 3 или 4 чисел [R, G, B, A]) - опционально
  * `meters` (число) - опционально
* **Запрос:** `{"id": 9, "cmd": "SET_FILAMENT_INFO", "lane": 0, "name": "ABS Black", "color": [0,0,0,255]}`

## Архитектура плагина Klipper (bmcu.py)

Klipper работает как хост, отправляя JSON-запросы в плату BMCU370. 
Плагин `bmcu.py` читает ответы, парсит JSON и обновляет внутренний объект Klipper `printer.objects.bmcu`.
Интерфейс Mainsail/Fluidd запрашивает этот объект через Moonraker API и отображает виджет, используя `bmcu_card.html`.
