# Логика управления филаментом BMCU370

В этом документе описана архитектура управления моторами и состояниями филамента, реализованная в классе `MMU_Logic` (`src/core/MMU_Logic.cpp`).

## Общая концепция

Каждый из 4 каналов (Lanes) независим и управляется собственной машиной состояний. 
Ключевая особенность BMCU370 (и систем типа AMS Lite) — использование **обратной связи по давлению (Pressure Feedback)**. Вместо точного отсчета шагов мотора, система поддерживает необходимое натяжение или слабину филамента (через датчик-переключатель или тензодатчик на входе). Это создает пружинящий буфер и позволяет экструдеру принтера самому тянуть пластик, в то время как MMU лишь "помогает" ему.

---

## 1. Состояния позиции филамента (Position State)

Хранятся в массиве `filament_now_position[4]`.

| Состояние | Описание |
|-----------|----------|
| `filament_idle` | Покой. Филамент не активен, но может поддерживаться слабое давление (натяг). |
| `filament_loading` | Загрузка. Подача филамента вперёд от катушки к хабу/экструдеру. |
| `filament_sending_out` | Переход в активное состояние печати. |
| `filament_using` | Активное использование (печать). Работает ПИД-регулятор давления. |
| `filament_pulling_back` | Обратная тяга (Retract). Возврат филамента после обрезки или при смене инструмента. |
| `filament_unloading` | Полная выгрузка филамента (пользовательский unload). |

---

## 2. Режимы движения мотора (`filament_motion_enum`)

Управляют непосредственным поведением мотора (`motors[i].motion`).

| Режим | Поведение |
|-------|-----------|
| `pressure_ctrl_idle` | Слабый PID по давлению. Поддерживает натяжение в покое. |
| `send` | Вращение вперёд с номинальной скоростью (загрузка). |
| `slow_send` | Медленное вращение вперёд (дожим). |
| `pull` | Вращение назад с номинальной скоростью (retract). |
| `pressure_ctrl_in_use` | Активный PID по давлению (отработка во время печати). |
| `velocity_control` | Управление скоростью/позицией (ограниченное перемещение на `target_distance`). |
| `stop` | Мотор обесточен. |

---

## 3. Управление давлением (Pressure Control)

Датчик давления читается через АЦП и выдает сырое напряжение (Raw Voltage) в диапазоне ~0–3.3В. Центральная точка ("нет давления") находится в районе **1.65В**.

В режиме `pressure_ctrl_in_use` работает следующий алгоритм (ПИД-регулятор):

1. **Если напряжение < 1.55В** — срабатывает гистерезис, сбрасываем флаг старого состояния.
2. **Если напряжение < 1.65В** — экструдер тянет пластик (натяжение). ПИД-регулятор дает команду мотору крутить **вперёд** (+), чтобы компенсировать натяжение и создать слабину.
3. **Если напряжение > 1.70В** — пластик уперся или экструдер делает retract (избыточное давление в буфере). ПИД-регулятор дает команду мотору крутить **назад** (-), чтобы выбрать слабину.

Таким образом, мотор MMU синхронизируется с мотором экструдера без прямой связи по данным, ориентируясь только на физическое натяжение прутка.

## 4. Жизненный цикл (Машина состояний)

Упрощенный переход состояний при смене филамента:

1. **[idle]**: Мотор остановлен. Ждем команду.
2. **Поступает команда LOAD (или FEED)** → Состояние меняется на **[sending_out]** (загорается зеленый LED). Мотор переходит в режим `send`.
3. **Пластик достигает экструдера** (сопротивление возрастает) → Переход в **[using]**. Мотор переходит в `slow_send` на 1.5 секунды для плотного прижима.
4. **Начало печати** → Переход в **[pressure_ctrl_in_use]**. Мотор динамически реагирует на давление, поддерживая его в диапазоне 1.65–1.70В.
5. **Поступает команда UNLOAD (или RETRACT)** → Переход в **[pulling_back]**. Мотор переходит в режим `pull` и отматывает фиксированную длину (сейчас жестко задано 200мм).
6. **Возврат в [idle]** после завершения дистанции.
