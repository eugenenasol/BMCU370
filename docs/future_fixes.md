# Будущие исправления (Future Fixes)

В этом файле описаны известные проблемы и планируемые доработки для прошивки BMCU370.

## 1. Исправление работы светодиодов (NeoPixel)
**Проблема:**
В текущем коде логика устанавливает цвета для светодиодов через `_hal->SetLED(...)` (который пишет в буфер NeoPixel), но нигде в цикле логики или HAL не вызывается `pixels.show()`. Из-за этого светодиоды могут не обновляться.

**План решения:**
- Добавить метод `UpdateLEDs()` (или `ShowLEDs()`) в интерфейс `I_MMU_Hardware`.
- Реализовать его в `BMCU_Hardware`, чтобы он вызывал `pixels.show()`.
- Вызывать этот метод в основном цикле `MMU_Logic::Run()` после обработки логики каналов.

## 2. Конфигурируемая дистанция возврата (Retract Distance)
**Проблема:**
В функции `Prepare_For_filament_Pull_Back` длина обратной тяги (retract) жестко зашита в коде (hardcoded):
`bool pulling = Prepare_For_filament_Pull_Back(200.0f);`
Если трубка Bowden на принтере длиннее или короче 200 мм, филамент либо не вернется полностью в катушку, либо мотор будет крутить вхолостую слишком долго.

**План решения:**
- Добавить сохраняемый параметр для длины retract (в `Flash_saves` или как настройку).
- Реализовать команду в протоколе (JSON Klipper), чтобы этот параметр можно было задавать через G-code / макросы Klipper (`bmcu_macros.cfg`).
- Заменить `200.0f` на значение из конфигурации.

## 3. Прочие улучшения
- Добавление полноценной обработки команд `LOAD` и `UNLOAD` в JSON-протокол.
- Timeout (Heartbeat): автоматическая остановка моторов при потере связи с Klipper.
- Вынос конфигурации инверсии моторов (`MOTOR_INVERT_CHx`) в сохраняемые во Flash параметры для возможности изменения без перекомпиляции.

## 4. Интеграция с Klipper Happy Hare (HH)
**Цель:**
Обеспечить полную совместимость и нативную работу BMCU370 как MMU устройства в экосистеме Happy Hare (HH).

**План решения:**
- Изучить требования протокола/макросов HH для работы со сторонними MMU (Enraged Rabbit Carrot Feeder, PicoMMU и т.д.).
- Настроить макросы загрузки/выгрузки (`LOAD`, `UNLOAD`, смена инструмента) для перехвата событий Klipper и корректной трансляции их в команды для BMCU370.
- Согласовать логику ПИД-подачи по давлению BMCU с алгоритмами синхронизации подачи Happy Hare (учитывая, что в BMCU мотор работает как ассистент, а не как жесткий податчик).
- Разработать необходимые конфигурационные файлы (`mmu_parameters.cfg` и т.д.) для интеграции платы.
