# Документация команд прошивки BMCU (Klipper JSON API)

В данном документе описаны все команды, поддерживаемые прошивкой BMCU через JSON-протокол. Взаимодействие происходит через отправку JSON-объектов.

## Формат запроса
Типовой запрос выглядит следующим образом:
```json
{
  "id": 123,
  "cmd": "COMMAND_NAME",
  "args": {
    "arg1": "value",
    "arg2": 42
  }
}
```
*Примечание: Поля `id` и `cmd` обязательны. `args` может быть как вложенным объектом, так и передаваться на верхнем уровне (хотя предпочтительнее вкладывать аргументы в `args`).*

---

## 1. Базовые команды и команды состояния

### `PING`
Проверка связи с платой.
* **Аргументы:** Нет
* **Ответ:** Содержит версию прошивки (`version`) и время работы в миллисекундах (`uptime`).

### `STATUS`
Получение полного состояния системы.
* **Аргументы:** Нет
* **Ответ:** Возвращает массив `lanes` с подробными данными для каждого канала (состояние датчиков, движение, включен ли автоподатчик, метры, давление, данные о филаменте), а также глобальные настройки ПИД-регулятора и параметров давления.

### `GET_SENSORS`
Получение сырого состояния оптических/механических датчиков.
* **Аргументы:** Нет
* **Ответ:** Массив `lane` (от 0 до 3), где 1 — датчик сработал (наличие филамента), 0 — датчик не сработал.

## 2. Управление движением

### `MOVE`
Команда для ручного управления моторами.
* **Аргументы:**
  * `axis` (String, обязательно): Ось для движения. Возможные значения: `"FEED"`, `"SELECTOR"`, либо индекс канала `"0"`, `"1"`, `"2"`, `"3"`.
  * `dist_mm` (Float, обязательно): Дистанция перемещения в миллиметрах. Отрицательные значения для ретракта.
  * `speed` (Float, обязательно): Скорость перемещения в мм/с.

### `STOP`
Экстренная остановка всех движений. Сбрасывает текущие активные действия.
* **Аргументы:** Нет

### `SELECT_LANE`
Выбор активного канала (филамента).
* **Аргументы:**
  * `lane` (Int, обязательно): Индекс канала от 0 до 3.

### `FEED_TO_EXTRUDER`
Автоматическая подача филамента до экструдера с контролем давления и определением упора.
* **Аргументы:**
  * `lane` (Int, по умолчанию -1): Канал для подачи.
  * `speed` (Float, по умолчанию 30.0): Скорость подачи в мм/с.
  * `max_mm` (Float, по умолчанию 250.0): Максимальная длина подачи, после которой операция будет прервана.
  * `pressure_thr` (Float, по умолчанию 0.20): Порог давления, при котором считается, что филамент уперся в экструдер.
  * `stall_ms` (Int, по умолчанию 300): Время (в мс), в течение которого должно сохраняться давление/остановка для фиксации упора.

## 3. Настройка и автоподача

### `SET_AUTO_FEED`
Управление режимом автоподачи (интеллектуального контроля натяжения/буфера) для канала.
* **Аргументы:**
  * `lane` (Int, опционально): Индекс канала (0-3). Если не указан, берется последний сохраненный канал.
  * `enable` / `ENABLE` (Bool, опционально): Включение (true) или выключение (false) автоподачи.
  * `overflow` / `OVERFLOW` (Bool, опционально): Флаг переполнения (для синхронизации LED индикации).

### `SET_PRESSURE_ADVANCED` (или `SET_PA`)
Установка расширенных параметров контроля давления. Все аргументы опциональны (применяются только те, что переданы). Аргументы можно передавать как в верхнем, так и в нижнем регистре.
* **Аргументы:**
  * `gain` / `GAIN` (Float): Коэффициент усиления давления (Gain).
  * `offset` / `OFFSET` (Float): Смещение давления (Offset).
  * `boost_thr` / `BOOST_THR` (Float): Порог срабатывания Boost-режима.
  * `boost_pwm` / `BOOST_PWM` (Float): PWM Boost-режима.
  * `boost_time` / `BOOST_TIME` (Int): Длительность Boost в миллисекундах.
  * `deadzone` / `DEADZONE` (Float): Мертвая зона для ретракта.
  * `min_pwm` / `MIN_PWM` (Float): Минимальный PWM для поддержания давления.
  * `tol` / `TOL` (Float): Допуск давления (Tolerance).

### `SET_MOVE_PID`
Настройка ПИД-регулятора для перемещения филамента.
* **Аргументы (опциональные, можно в любом регистре):**
  * `p` / `P` (Float)
  * `i` / `I` (Float)
  * `d` / `D` (Float)
  * `zero` / `ZERO` (Float): Нулевое значение PWM для мотора.

### `CALIBRATE`
Калибровка нулевого значения давления для тензодатчиков.
* **Аргументы:**
  * `lane` (Int, опционально): Канал для калибровки (0-3). Если не указан, калибруются все каналы сразу.
* **Ответ:** Возвращает новые откалиброванные нулевые значения для указанного канала или всех каналов.

## 4. Управление данными о филаментах (RFID/Metadata)

### `GET_FILAMENT_INFO`
Получение метаданных филамента на конкретном канале.
* **Аргументы:**
  * `lane` (Int, обязательно): Канал (0-3).
* **Ответ:** Содержит `meters`, `pressure`, `rfid`, `name`, `temp_min`, `temp_max` и `color`.

### `SET_FILAMENT_INFO`
Запись или обновление метаданных филамента.
* **Аргументы:**
  * `lane` (Int, обязательно): Канал (0-3).
  * `id_str` (String, опционально, макс. 8 символов): RFID или идентификатор катушки.
  * `name` (String, опционально, макс. 20 символов): Название материала (например, "PETG Black").
  * `temp_min` (Int, опционально): Минимальная температура печати.
  * `temp_max` (Int, опционально): Максимальная температура печати.
  * `color` (Array, опционально): Массив RGBA из 3 или 4 целых чисел `[R, G, B, A]`.
  * `meters` (Float, опционально): Метраж израсходованного или оставшегося филамента.

## 5. Диагностика

### `TEST_MOTOR`
Запуск диагностического прогона мотора (подача сырого PWM).
* **Аргументы:**
  * `lane` (Int, обязательно): Канал мотора (0-3).
  * `pwm` (Int, обязательно): Значение мощности (обычно от -255 до 255).
  * `duration` (Int, опционально, по умолчанию 1000): Время в миллисекундах для теста.
