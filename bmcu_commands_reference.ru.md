# Справочник команд BMCU (Klipper + JSON)

В этом документе приведено полное описание всех команд, реализованных в текущей версии интеграции BMCU для Klipper.

## 1. Системные и диагностические команды

### BMCU_PING
Проверка связи и получение базовой информации о системе.
* **G-code:** `BMCU_PING [WAIT=0.0]`
* **JSON:** `{"cmd": "PING"}`
* **Ответ:** Содержит `version` (версия прошивки) и `uptime` (время работы в мс).

### BMCU_STATUS
Запрос полного состояния всех 4-х каналов.
* **G-code:** `BMCU_STATUS`
* **JSON:** `{"cmd": "STATUS"}`
* **Ответ:** Массив `lanes` с данными по каждому каналу: датчики, движение, давление, пробег, RFID и т.д.

### BMCU_GET_SENSORS
Быстрый опрос состояния датчиков наличия филамента.
* **G-code:** `BMCU_GET_SENSORS [WAIT=0.5]`
* **JSON:** `{"cmd": "GET_SENSORS"}`
* **Ответ:** Массив `lane` (0 или 1 для каждого канала).

### BMCU_TEST_MOTOR
Прямое управление мотором для диагностики (ШИМ).
* **G-code:** `BMCU_TEST_MOTOR LANE=0 PWM=150 DURATION=1000`
* **JSON:** `{"cmd": "TEST_MOTOR", "args": {"lane": 0, "pwm": 150, "duration": 1000}}`

## 2. Команды управления движением

### BMCU_MOVE
Универсальное движение по осям.
* **G-code:** `BMCU_MOVE AXIS=0 DIST=50 SPEED=20 [WAIT=0.0]`
* **JSON:** `{"cmd": "MOVE", "args": {"axis": "0", "dist_mm": 50, "speed": 20}}`
* **Параметр AXIS:**
  * `"0"` - `"3"`: Конкретный канал (мотор катушки).
  * `"FEED"`: Мотор текущего активного канала.
  * `"SELECTOR"`: Мотор селектора (если установлен).
  * `"SPOOL0"` - `"SPOOL3"`: Альтернативное обращение к катушкам.

### BMCU_STOP
Экстренная остановка всех процессов и моторов.
* **G-code:** `BMCU_STOP`
* **JSON:** `{"cmd": "STOP"}`

### BMCU_SELECT_LANE
Программный выбор активного канала.
* **G-code:** `BMCU_SELECT_LANE LANE=0 [WAIT=0.3]`
* **JSON:** `{"cmd": "SELECT_LANE", "args": {"lane": 0}}`

### BMCU_FEED_TO_EXTRUDER
"Умная" подача филамента до головы принтера.
* **G-code:** `BMCU_FEED_TO_EXTRUDER LANE=-1 SPEED=30 MAX_MM=250 PRESSURE_THR=0.20 STALL=300`
* **JSON:** `{"cmd": "FEED_TO_EXTRUDER", "args": {"lane": -1, "speed": 30.0, "max_mm": 250.0, "pressure_thr": 0.2, "stall_ms": 300}}`
* **Логика остановки:** Команда завершится успешно при скачке давления (филамент уперся) или при детекции пропуска шагов (Stall).

## 3. Настройка и калибровка

### BMCU_SET_PA (или BMCU_SET_PRESSURE_ADVANCED)
Тонкая настройка системы активного поддержания давления филамента. Все параметры опциональны; если параметр не указан, сохраняется его текущее значение в памяти.

* **G-code:** `BMCU_SET_PA [ПАРАМЕТРЫ]`
* **JSON:** `{"cmd": "SET_PRESSURE_ADVANCED", "args": {"gain": ..., "offset": ..., ...}}`

**Список параметров:**
*   `GAIN` (Float): Основной коэффициент усиления. Определяет агрессивность реакции мотора на изменение давления. Чем выше значение, тем быстрее мотор будет пытаться компенсировать просадку.
*   `OFFSET` (Float): Смещение целевого давления (в барах). Позволяет программно сдвинуть «ноль» или задать базовый преднатяг.
*   `TOL` (Float): Допуск (Tolerance). Ошибка давления, которую система считает допустимой и не пытается корректировать. Предотвращает постоянное «дрожание» мотора при минимальных колебаниях.
*   `MIN_PWM` (Float): Минимальная мощность (0-1000). Минимальное заполнение ШИМ, которое подается на мотор для преодоления трения покоя.
*   `DEADZONE` (Float): Мертвая зона ретракта (в мм). Дистанция, на которую филамент должен вернуться назад, прежде чем система начнет активно сбрасывать давление.
*   `BOOST_THR` (Float): Порог активации Boost (в барах). Если разница между целевым и текущим давлением превышает этот порог, включается форсированный режим.
*   `BOOST_PWM` (Float): Добавочная мощность Boost (0-1000). Добавляется к текущему ШИМ при срабатывании порога Boost для резкого рывка мотора.
*   `BOOST_TIME` (Int): Длительность импульса Boost (в миллисекундах). Определяет, как долго будет длиться форсированный толчок.

### BMCU_CALIBRATE [LANE=<0-3>] [WAIT=<seconds>]
Запускает процесс калибровки нуля давления.
- Если `LANE` не указан, калибруются все пустые линии.
- **Важно**: Эта команда автоматически сохраняет новые значения нулей во Flash-память, не затрагивая остальные настройки в RAM.

### BMCU_SAVE_PARAMS [WAIT=<seconds>]
Сохраняет все текущие настройки из оперативной памяти (PID, PA, Нули) в энергонезависимую Flash-память.

### BMCU_LOAD_PARAMS [WAIT=<seconds>]
Загружает настройки из Flash-памяти, перезаписывая текущие значения в RAM. Полезно для отката неудачных настроек.

### BMCU_SET_AUTO_FEED LANE=<0-3> ENABLE=<0|1> [OVERFLOW=<0|1>]
Включает/выключает режим автоматической подачи.
- **Примечание**: Состояние этого режима не сохраняется во Flash и всегда выключено при старте.
* **JSON:** `{"cmd": "SET_AUTO_FEED", "args": {"lane": 0, "enable": true, "overflow": false}}`

### BMCU_SET_MOVE_PID
Настройка PID-регулятора для перемещений.
* **G-code:** `BMCU_SET_MOVE_PID P=10.0 I=0.1 D=1.0 ZERO=15`
* **JSON:** `{"cmd": "SET_MOVE_PID", "args": {"p": 10.0, "i": 0.1, "d": 1.0, "zero": 15}}`

### BMCU_CALIBRATE
Определение "нулевого" значения датчиков давления.
* **G-code:** `BMCU_CALIBRATE [LANE=0]`
* **JSON:** `{"cmd": "CALIBRATE", "args": {"lane": 0}}`
* *Если LANE не указан, калибруются все 4 канала.*

## 4. Метаданные филамента

### BMCU_SET_FILAMENT_INFO
Запись данных о катушке в память BMCU.
* **G-code:** `BMCU_SET_FILAMENT_INFO LANE=0 NAME="PLA Red" COLOR="#FF0000" ID="XYZ123" METERS=120.5`
* **JSON:** `{"cmd": "SET_FILAMENT_INFO", "args": {"lane": 0, "name": "...", "color": [255,0,0,255], "id_str": "...", "meters": 120.5}}`

### BMCU_GET_FILAMENT_INFO
Чтение данных о катушке.
* **G-code:** `BMCU_GET_FILAMENT_INFO LANE=0`
* **JSON:** `{"cmd": "GET_FILAMENT_INFO", "args": {"lane": 0}}`

---

## 5. Прямой вызов (BMCU_CALL)
Для отладки новых команд, не имеющих своего G-code макроса.
* **G-code:** `BMCU_CALL CMD=Команда ARGS='{"ключ": "значение"}'`
* **Пример:** `BMCU_CALL CMD=STATUS WAIT=0.5`

---

## 6. Пользовательские макросы (bmcu_macros.cfg)

Эти макросы определены в файле `bmcu_macros.cfg` и предназначены для удобного вызова из интерфейса (Mainsail/Fluidd).

### Глобальные операции
*   **`BMCU_STOP_ALL`**: Вызов экстренной остановки `BMCU_STOP`.
*   **`BMCU_SHOW_STATUS`**: Вывод закешированного статуса в консоль (`BMCU_STATUS`).
*   **`BMCU_FETCH_STATUS`**: Запрос свежих данных у платы (`BMCU_CALL CMD=STATUS WAIT=0.5`).
*   **`BMCU_REFRESH`**: Полное обновление данных (аналог `FETCH_STATUS`, но с ожиданием 1.0 сек).

### Операции по каналам (L0 - L3)
Для каждого канала `X` (0, 1, 2, 3) созданы следующие макросы:
*   **`BMCU_SELECT_LX`**: Выбрать канал как активный.
*   **`BMCU_FEED_LX`**: Подать филамент на дистанцию по умолчанию.
*   **`BMCU_RETRACT_LX`**: Выполнить ретракт на дистанцию по умолчанию.
*   **`BMCU_AUTO_ON_LX`**: Включить режим автоподачи (поддержание давления).
*   **`BMCU_AUTO_OFF_LX`**: Выключить режим автоподачи.

### Сложные макросы
*   **`TEST_FTE`**: Тестовый макрос для команды `FEED_TO_EXTRUDER`. Выполняет подачу и выводит в консоль Klipper причину остановки (давление, застревание или превышение дистанции).
